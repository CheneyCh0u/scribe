# 004 — UI 交互方案（三选一）

状态：已定稿（2026-07-17）。结论见文末；定稿设计与权威 token 表在 [ui/scribe-ui.html](ui/scribe-ui.html)。

## 背景

主面板 + 设置页需要确定视觉与交互方向。三套方案在**布局结构、信息密度、色温**三个维度拉开差异，分别对应同品类三个已验证形态。HTML 原型见 [ui/options.html](ui/options.html)（含各方案完整 design token 表，用于 SwiftUI 1:1 还原）。

## 同类调研

- **Maccy**：贴菜单栏的极简紧凑列表，最快最轻，图片只有小缩略图。
- **Raycast 剪贴板**：居中双栏命令面板，左列表右预览，键盘优先，⌘1-9 快选。
- **Paste**：屏幕底部横向大卡片架，图片体验最好，偏鼠标操作。

## 三套方案

| | A · 系统原生 | B · 指挥台 | C · 置物架 |
|---|---|---|---|
| 论点 | 像系统自带组件 | 键盘唯一输入，预览消灭二次操作 | 剪贴板是可视实物 |
| 形态 | 368pt 紧凑单列，就近弹出 | 760×440 居中双栏（列表+预览） | 底部全宽卡片架，172×168 卡片 |
| 材质/色 | 浅色毛玻璃 + 系统强调色 | 石墨暗色 + 铜橙 #E3A25C | 暖纸色 + 朱砂 #C9502E，宋体组标签 |
| 强项 | 最轻、零学习成本、实现最快 | 长文本/代码/图片原地看全貌、键盘效率最高 | 图片体验最强、视觉记忆点 |
| 弱项 | 长内容只有两行摘要 | 面板大、存在感强 | 占满底边、可见条数少、封面解码有内存压力 |

共享原语 token（三案通用）：spacing 4/6/8/12/16/20；字号 11/12/13/15；motion 120/180ms `cubic-bezier(0.16,1,0.3,1)`；字体 SF Pro + PingFang SC，代码 SF Mono。

## 推荐

**B 指挥台**：与"快捷键唤起、键盘优先"的产品论点最一致，右栏预览免去"粘贴出来才知道对不对"的二次操作，⌘1-9 常显徽标压缩高频路径；双栏实现成本可控（选中态驱动预览视图）。更看重"轻、像系统组件"则选 A；C 与内存红线和键盘优先定位有张力。

## 结论

用户选定：**A 系统原生材质 × B 双栏布局**（左列表、右详情），并追加两点要求：

1. **深浅双模式随系统切换**，窗口不覆盖 appearance；系统语义色/系统材质自动适配，自定义色在 Asset Catalog 提供 Any/Dark 双值。
2. **材质、色值、透明度按 macOS 聚焦（Spotlight）实拍取样**：近白高透重模糊面板（浅 rgba(252,251,250,0.82)/深 rgba(30,30,34,0.80) + blur38）、大圆角（面板 22 / 行 10）、**选中行为安静灰填充（黑 7.5% / 白 9%），不用强调色反白**、筛选 chips 为白色胶囊 + 细阴影、搜索区无框 16pt、选中行右侧显示操作提示 + 白色按键帽。强调色仅用于链接/焦点环/设置选中标签。

Token 权威来源：[ui/tokens.md](ui/tokens.md)；`ui/scribe-ui.html` 为视觉原型（与 tokens.md 同步维护）；`ui/options.html` 保留作方案探索记录。

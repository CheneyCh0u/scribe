# 006 — 状态栏与应用图标方向

状态：已选定并实施（2026-07-17）

## 背景

Scribe 已完成主要功能与界面，但仍使用 SF Symbols 的 `doc.on.clipboard` 作为状态栏临时图标，也没有正式应用图标。本轮先设计视觉方向，不直接生成生产资产。

图标需要同时满足两种截然不同的使用环境：

- **状态栏图标**：约 18pt，作为 AppKit template image 使用，只能依赖黑色、透明与 alpha，在深浅菜单栏中自动着色。
- **应用图标**：1024×1024 分层彩色图标，需要在 Dock、Finder、设置与搜索结果的小尺寸下保持辨识度。

## 共同约束

1. 状态栏与应用图标共享同一母题，但不机械缩放同一张图。
2. 状态栏 glyph 不使用细线、文字、颜色或依赖阴影的细节。
3. 应用图标不复刻应用界面，不把系统剪贴板符号直接装进圆角方块。
4. 严格延续现有 UI token：Spotlight 式系统材质、安静灰阶、系统蓝克制点缀、大圆角。
5. 选型原型使用原生 HTML/CSS 与内联 SVG，不引入外部资源。

## 五套方向

### A · 层叠历史

三层圆角内容面板表现历史堆叠，最直接地表达“剪贴板历史”。系统蓝仅作为当前条目的小型状态点。

### B · 回填

两层内容面板加一条回转箭头，表达从历史记录回到当前应用的核心动作。

### C · 时间切片

三条横向记录与一条时间轴组合，突出按时间分组、最近使用和历史召回。

### D · 搜索聚焦

搜索环覆盖在安静的历史行上，表达 Scribe 的“唤起即搜索”路径，并借用现有界面的无框搜索语言。

### E · 面板唤起

一个非激活双栏面板从背景中浮起，强调全局快捷键唤起和左列表、右预览的产品特征。

## 选型方式

视觉原型见 [ui/icon-options.html](ui/icon-options.html)。选定方向后再：

1. 按 16/18/32pt 光学校正状态栏 SVG；
2. 生成 1024px 应用图标母版及 macOS 所需尺寸；
3. 接入 Asset Catalog，并将状态栏临时 SF Symbol 替换为 template image；
4. 在浅色、深色菜单栏及 Dock/Finder 中真机验收。

## 结论

用户选定 **B · 回填**。已实施：

- `AppIcon.appiconset`：16、32、64、128、256、512、1024px 全尺寸 PNG；
- `MenuBarIcon.imageset`：18pt 矢量 template glyph；
- AppDelegate 优先加载自定义状态栏图标，资产缺失时回退到 `doc.on.clipboard`；
- Xcode 工程指定 `AppIcon` 为应用图标集。

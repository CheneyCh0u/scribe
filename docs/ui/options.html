Scribe · UI 交互方案对比

三套方案在布局结构、信息密度、色温三个维度真正拉开差异，分别对应同品类三个被验证过的形态。每套含主面板、设置页与独立 design token 表，选定后按 token 在 SwiftUI 中 1:1 还原。悬停可看交互状态。

A

系统原生

横向对比

A · 系统原生B · 指挥台C · 置物架
唤起形态就近小窗（菜单栏/光标处）屏幕居中大面板屏幕底部整条升起
单屏可见条数约 8 条约 9 条 + 完整预览约 7 张卡
长文本/代码体验两行摘要，需粘贴后看右栏原地看全文卡片约 6 行
图片体验44pt 小缩略图列表小图 + 右栏大图整卡封面，最强
键盘效率高最高（⌘1-9 常显 + 预览免二次操作）中（横向方向键）
存在感/打扰度最低中高（占满底边）
实现风险最低低（双栏联动）中（横向虚拟化+封面解码对内存红线压力大）

我的推荐：B 指挥台。产品论点是"快捷键唤起、键盘优先"，B 的右栏预览让长文本、代码、图片都无需二次操作即可确认内容，⌘1-9 常显徽标把高频路径压缩到一次按键；实现上双栏只是列表选中态驱动预览视图，成本可控。若你更看重"轻、像系统组件"，选 A；C 视觉最讨喜但与内存红线和键盘优先的定位有张力。

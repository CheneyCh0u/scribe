# 开发流程与文档维护规则

本文件是流程类规则的唯一权威来源。CLAUDE.md 只保留项目现状摘要和到这里的索引。

## 文档体系分工

| 文件 | 性质 | 更新方式 |
|---|---|---|
| `CLAUDE.md` | 项目**当前现状**摘要（活文档） | 随时修正，永远与代码一致，保持精简 |
| `docs/NNN-主题.md` | **历史设计决策**（日志） | 只增不改；被推翻时在文首标注「状态：已被 NNN 取代」 |
| `docs/process.md` | 流程与维护规则（本文件） | 流程变化时修订 |

## 维护规则

1. **CLAUDE.md 永不过时**：任何改变技术栈、架构、目录结构、约定的变更，必须在同一次提交里同步更新 CLAUDE.md 对应段落；发现它与代码不符时，先修正它再继续干活。
2. **CLAUDE.md 保持小**：只放摘要、表格、索引链接；细节一律下沉到 docs/ 里对应文档。
3. **每次设计先写 docs**：新功能/重构在动代码前，先在 `docs/` 新增 `NNN-主题.md`（编号递增），记录背景、决策、被否方案，然后才实现。
4. **已实施的设计文档不删除、不改写历史**；方案被推翻时新增文档并在旧文档标注取代关系。
5. **UI token 单一权威**：`docs/ui/tokens.md` 是 UI token 的唯一权威来源。任何视觉调整先改 tokens.md，再同步原型 `docs/ui/scribe-ui.html` 与 Swift 实现（`DesignTokens.swift` + Asset Catalog）；禁止在视图代码里散写色值/尺寸魔法数字。
6. **Todolist 必须随实现同步维护**：每完成一个功能，在同一次提交里把对应 todolist 条目勾选为 `[x]`（当前 todolist 在 [003-features.md](003-features.md)）；整个阶段完成后同步更新 CLAUDE.md 的「当前进度」。勾选 checkbox 是状态更新，不算改写历史，是"只增不改"规则的唯一豁免。实现中新冒出的任务追加进 todolist，不允许只干活不记账。

## 项目约定

- **性能红线**：面板冷启动唤起 <100ms；空闲 CPU ≈0%；常驻内存 <50MB；图片永不整批载入内存，缩略图懒加载。
- **隐私**：`TransientType` 内容始终不记录；`ConcealedType`（密码类）默认记录但列表打码显示，用户可在设置中关闭记录（决策见 003）。
- **语言**：提交信息、代码注释用英文；文档用中文。

## 实施节奏

按 [002-architecture.md](002-architecture.md) 的四个阶段推进，每阶段独立可用、可合并；阶段完成后更新 CLAUDE.md 的「当前进度」。
